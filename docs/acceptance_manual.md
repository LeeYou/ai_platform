## 1. 验收概述

### 1.1 验收目的

本手册旨在为 AI 平台的交付验收提供标准化的测试项目、操作步骤及通过标准。通过系统化的验收测试，确认平台各子系统功能完整、接口可用、业务流程贯通，满足合同约定的交付条件。

### 1.2 验收范围

本次验收涵盖 AI 平台以下子系统：

| 子系统             | 服务端口 | 说明                                   |
|--------------------|----------|----------------------------------------|
| 授权服务（License） | 8003     | 密钥对、客户、授权许可管理             |
| 训练服务（Train）   | 8001     | 训练任务管理、模型导出                 |
| 测试服务（Test）    | 8002     | 单条测试、批量测试、模型版本对比       |
| 编译服务（Build）   | 8004     | C++ 编译、能力列表                     |
| 生产推理服务（Prod）| 8080     | 推理接口、健康检查、管理热加载、编排引擎 |
| 生产 Web 前端       | —        | Dashboard、ApiTest、Status、Admin、Pipelines、PipelineEdit、PipelineTest |

### 1.3 验收方法

1. **逐项执行**：按照本手册中每一条测试项的操作步骤依次执行。
2. **记录结果**：在每项测试完成后，记录实际结果并与预期结果进行比对。
3. **判定通过**：实际结果满足"通过标准"列所述条件即为该项通过。
4. **汇总结论**：全部测试项执行完毕后，填写验收结论模板，由双方签字确认。

### 1.4 术语说明

| 术语          | 说明                                           |
|---------------|------------------------------------------------|
| API           | 应用程序编程接口                               |
| Health Check  | 服务健康检查端点，用于确认服务运行状态         |
| Hot Reload    | 热加载，无需重启服务即可重新加载配置或模型     |
| Pipeline      | AI 编排流水线，由多个处理节点组成的推理工作流  |
| Admin Token   | 管理员令牌，用于保护管理类接口的鉴权凭证       |
