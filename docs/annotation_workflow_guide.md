## 2. 前置准备

### 2.1 启动平台

```bash
# 初始化宿主机目录（仅首次部署执行）
cd deploy/mount_template
bash init_host_dirs.sh

# 启动开发环境所有服务
cd deploy
docker compose up -d --build

# 验证服务状态
docker compose ps
```

### 2.2 准备样本数据

将原始样本图片放入宿主机数据集目录：

```bash
# 数据集根目录
/data/ai_platform/datasets/

# 示例：为人脸检测准备数据集
/data/ai_platform/datasets/face_detect/
├── train/
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
└── val/
    ├── img_101.jpg
    └── ...
```

> **注意:** 数据集目录在容器内映射为 `/workspace/datasets`（只读挂载）。标注导出结果会写入数据集子目录。

### 2.3 确认 AI 能力已注册

在训练 Web 页面（`http://<IP>:8001`），进入 **AI能力** 页签，确认目标 AI 能力已创建。例如：
- `face_detect`（人脸检测）
- `recapture_detect`（翻拍检测）
- `id_card_classify`（证件分类）
- `handwriting_reco`（手写识别）

如果没有，请先创建 AI 能力条目。

## 3. 第一步：样本标注

### 3.1 访问标注页面

在浏览器中打开训练服务：

```
http://<服务器IP>:8001
```

点击左侧导航栏的 **「样本标注」** 菜单项，进入标注项目列表页。

> 对应前端路由：`http://<服务器IP>:8001/#/annotations`

### 3.2 创建标注项目

点击 **「新建标注项目」** 按钮，填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **项目名称** | 标注项目的名称 | `人脸检测v2标注` |
| **关联AI能力** | 下拉选择已注册的 AI 能力 | `face_detect` |
| **标注类型** | 选择适合的标注工具类型 | 见下表 |
| **神经网络选型** | 后续训练使用的网络架构 | `yolov8` |
| **数据集路径** | 容器内数据集路径 | `/workspace/datasets/face_detect/train` |
| **标签配置** | JSON 格式的标签定义 | 见下方各类型示例 |

#### 标注类型对照表

| 标注类型 | 适用场景 | 标签配置示例 |
|---------|---------|------------|
| **二分类** (`binary_classification`) | 活体检测、屏幕攻击检测、翻拍检测 | `{"labels": ["真人", "假体"]}` |
| **多分类** (`multi_classification`) | 证件分类、场景分类、表情识别 | `{"labels": ["身份证", "驾照", "护照", "银行卡"]}` |
| **目标检测** (`object_detection`) | 人脸检测、车牌检测、YOLO检测 | `{"labels": ["face", "plate"]}` |
| **OCR文字识别** (`ocr`) | 车牌号识别、合同字段、文字区域 | `{"labels": ["text"]}` |
| **图像分割** (`segmentation`) | 语义分割、实例分割 | `{"labels": ["person", "background"]}` |

#### 神经网络选型参考

| AI能力类型 | 推荐网络 | 说明 |
|-----------|---------|------|
| 二分类/多分类 | `resnet18` / `mobilenetv2` | 轻量级分类网络 |
| 目标检测 | `yolov8` / `yolov5` | 实时检测网络 |
| OCR | `crnn` / `ppocr` | 文字识别网络 |
| 图像分割 | `unet` / `deeplabv3` | 分割网络 |

### 3.3 进入标注工作台

创建项目后，在项目列表中点击 **「开始标注」** 按钮，或直接点击项目名称链接，进入标注工作台。

> 对应前端路由：`http://<服务器IP>:8001/#/annotations/<项目ID>`

### 3.4 标注操作

标注工作台分为左右两栏：
- **左侧**：图片显示区域，底部有翻页导航
- **右侧**：标注工具面板（根据标注类型自动切换）

#### 3.4.1 二分类标注

1. 查看左侧图片
2. 在右侧点击对应的标签按钮（如 **「真人」** 或 **「假体」**）
3. 系统自动保存并跳转到下一张未标注图片
4. **快捷键**：按 `0` 选第一个标签，按 `1` 选第二个标签

#### 3.4.2 多分类标注

1. 查看左侧图片
2. 在右侧点击对应的分类标签按钮
3. 自动保存并跳转
4. **快捷键**：按数字键 `0`-`9` 对应各个标签

#### 3.4.3 目标检测标注

1. 在左侧图片上 **拖拽鼠标画矩形框** 标记目标区域
2. 松开鼠标后，在弹出的下拉框中选择类别标签
3. 重复添加多个检测框
4. 右侧面板显示已标注的框列表，可点击 **删除** 移除错误标注
5. 点击 **「保存标注」** 按钮保存当前图片标注
6. 使用 **←** / **→** 导航到下一张

#### 3.4.4 OCR 标注

1. 在左侧图片上 **点击多个点形成多边形区域** 围出文字区域
2. 在右侧输入框中填写该区域的文字内容
3. 可添加多个文字区域
4. 点击 **「保存标注」** 按钮保存

#### 3.4.5 通用操作

| 操作 | 方式 |
|------|------|
| 上一张 | 点击 **◀** 按钮 或 按 **←** 方向键 |
| 下一张 | 点击 **▶** 按钮 或 按 **→** 方向键 |
| 跳转到指定图片 | 在跳转输入框中输入编号并回车 |
| 筛选样本 | 下拉选择 **全部** / **未标注** / **已标注** |
| 查看进度 | 顶部进度条显示 `已标注数 / 总样本数` |

### 3.5 导出标注结果

标注完成后（或部分标注），回到标注项目列表页，点击对应项目的 **「导出」** 按钮。

系统自动将标注结果导出到数据集目录下的 `annotations_export/` 子目录。

| 标注类型 | 导出格式 | 导出目录结构 |
|---------|---------|-------------|
| 二分类/多分类 | 分类目录结构 | `annotations_export/真人/`, `annotations_export/假体/`, `labels.json` |
| 目标检测 | YOLO 格式 | `annotations_export/images/`, `annotations_export/labels/`, `data.yaml` |
| OCR | 文本坐标格式 | `annotations_export/images/`, `annotations_export/labels/` |
| 其他 | JSON 格式 | `annotations_export/annotations.json` |

> **导出路径示例：** 若数据集路径为 `/workspace/datasets/face_detect/train`，则导出到 `/workspace/datasets/face_detect/train/annotations_export/`

## 4. 第二步：模型训练

### 4.1 访问训练页面

在浏览器中打开：

```
http://<服务器IP>:8001
```

点击左侧导航栏的 **「训练任务」** 菜单项。

> 对应前端路由：`http://<服务器IP>:8001/#/jobs`

### 4.2 创建训练任务

点击 **「新建训练任务」** 按钮：

| 字段 | 说明 | 示例 |
|------|------|------|
| **AI 能力** | 选择要训练的 AI 能力（与标注项目关联同一能力） | `face_detect` |
| **版本号** | 模型版本标识 | `v2.0` |

> **关键联动**：创建训练任务时选择的 AI 能力与上一步标注项目中选择的 AI 能力一致，确保标注的数据用于对应的训练任务。

### 4.3 监控训练进度

- 训练任务提交后，状态依次为：`pending` → `running` → `done` / `failed`
- 点击任务行可查看：
  - **训练日志**：实时滚动输出
  - **训练指标**：loss 变化、准确率等
- 支持操作：**暂停** / **恢复** / **停止** 训练

### 4.4 训练产物

训练完成后，模型文件保存在：

```
宿主机：/data/ai_platform/models/<capability_name>/<version>/
容器内：/workspace/models/<capability_name>/<version>/
```

可在 **「模型管理」**（`http://<IP>:8001/#/models`）页面查看所有已训练的模型。

## 5. 第三步：模型测试

### 5.1 访问测试页面

在浏览器中打开：

```
http://<服务器IP>:8002
```

### 5.2 单图测试

进入 **「单图测试」** 页面：

1. 选择 AI 能力和模型版本
2. 上传一张测试图片
3. 点击 **推理** 查看结果
4. 结果包括可视化展示和原始 JSON 输出

### 5.3 批量测试

进入 **「批量测试」** 页面：

1. 选择 AI 能力和模型版本
2. 输入测试数据集路径（容器内路径，如 `/workspace/datasets/face_detect/val`）
3. 提交批量推理任务
4. 实时进度条（WebSocket 推送）
5. 完成后查看准确率、召回率等指标报告

### 5.4 版本对比

进入 **「模型对比」** 页面：

1. 选择同一 AI 能力的两个不同版本
2. 设置对比样本数
3. 并排查看两个版本的推理差异

> 如测试结果不满意，**返回第一步**修正标注或增加样本，重新训练。

## 6. 第四步：授权管理

### 6.1 访问授权页面

在浏览器中打开：

```
http://<服务器IP>:8003
```

### 6.2 创建客户密钥对

进入 **「密钥管理」** 页面：

1. 点击 **「生成密钥对」**
2. 填写客户标识和私钥输出路径（如 `/data/licenses/keys/customer_a/private_key.pem`）
3. 生成的密钥对用于后续编译时对推理库进行签名绑定

### 6.3 生成授权证书

进入 **「授权证书」** 页面：

1. 选择客户密钥对
2. 配置授权参数（到期时间、绑定硬件等）
3. 生成授权证书文件（`.lic`）

## 7. 第五步：推理库编译

### 7.1 访问编译页面

在浏览器中打开：

```
http://<服务器IP>:8004
```

### 7.2 创建编译任务

进入 **「新建编译」** 页面：

1. **选择 AI 能力**：下拉选择（与标注/训练关联同一能力）
2. **选择平台**：`linux_x86_64` / `linux_aarch64` / `windows_x86_64`
3. **选择编译类型**：`release` / `debug`
4. **绑定客户密钥对**：从下拉列表选择（关联授权服务）
5. **额外 CMake 参数**（可选）：如 `-DENABLE_GPU=ON`
6. 点击 **「提交编译」**

### 7.3 编译产物

编译完成后，可在 **「编译历史」** 页面下载产物：

- `libai_<capability>.so`（Linux 动态库）或 `.dll`（Windows）
- `libai_runtime.so`（运行时库）
- 模型文件（加密后的 `.bin`）
- 头文件（`ai_api.h`）

产物同时保存在宿主机：

```
/data/ai_platform/libs/<arch>/<capability>/
```

## 9. 第七步：微调迭代（版本迭代）

当需要提升模型精度或扩展新场景时，按以下流程迭代：

### 9.1 迭代流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        微调迭代流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 补充/修正样本数据                                             │
│     ↓                                                           │
│  2. 在标注页面(:8001/#/annotations)创建新的标注项目                  │
│     → 关联同一AI能力，可选择不同或相同的网络选型                       │
│     → 使用新版本标识（如 v2.1）                                    │
│     ↓                                                           │
│  3. 完成新样本标注 & 导出                                          │
│     ↓                                                           │
│  4. 在训练页面(:8001/#/jobs)创建新训练任务                           │
│     → 选择同一AI能力，使用新版本号                                   │
│     ↓                                                           │
│  5. 在测试页面(:8002)对比新旧版本                                   │
│     → 使用「模型对比」功能确认改进效果                                │
│     ↓                                                           │
│  6. 确认提升后，重新编译推理库(:8004)                                │
│     → 选择新版本模型                                               │
│     ↓                                                           │
│  7. 更新生产服务(:8080)中的模型                                     │
│     → 热更新或重启容器                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 9.2 版本管理建议

| 版本号格式 | 适用场景 | 示例 |
|-----------|---------|------|
| `v1.0` → `v1.1` | 小幅标注修正，补充少量样本 | 修复误标注后重训 |
| `v1.0` → `v2.0` | 大幅增加样本，切换网络架构 | 从 ResNet 切换到 MobileNet |
| `v1.0-finetune` | 基于已有模型微调 | 在特定场景数据上微调 |

### 9.3 迭代检查清单

- [ ] 新样本已放入数据集目录
- [ ] 标注项目已创建并关联正确的 AI 能力
- [ ] 标注完成度达标（建议 >95%）
- [ ] 标注已导出为训练兼容格式
- [ ] 新版本训练任务已完成
- [ ] 新版本测试指标优于旧版本
- [ ] 推理库已重新编译
- [ ] 生产环境已更新

## 常见问题

### Q: 标注页面打不开？

**A:** 确认训练服务已启动：

```bash
docker compose ps | grep train
# 应显示 ai-train 状态为 running (healthy)
```

浏览器访问 `http://<IP>:8001`，左侧导航栏点击 **「样本标注」**。

### Q: 标注项目看不到图片？

**A:** 检查数据集路径是否正确：

```bash
# 查看容器内数据集
docker exec ai-train ls /workspace/datasets/<你的数据集路径>/
```

确保路径中有支持的图片格式（`.jpg`, `.jpeg`, `.png`, `.bmp`, `.tiff`, `.webp`, `.gif`）。

### Q: 导出后在哪里找标注文件？

**A:** 导出文件在数据集的 `annotations_export/` 子目录：

```bash
# 宿主机查看
ls /data/ai_platform/datasets/<capability>/<子路径>/annotations_export/
```

### Q: 训练任务一直 pending？

**A:** 检查 Celery Worker 和 Redis 连接：

```bash
docker exec ai-train celery -A tasks.celery_app inspect active
docker exec ai-train redis-cli -h redis ping
```

### Q: 如何确认 GPU 可用？

```bash
docker exec ai-train python3 -c "import torch; print(torch.cuda.is_available())"
```
