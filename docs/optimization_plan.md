## 1. CUDA 版本兼容性分析

### 1.1 用户已有镜像

| 镜像 | 用途 | CUDA 版本 |
|------|------|-----------|
| `nvidia/cuda:11.8.0-cudnn8-devel-ubuntu22.04` | 训练（含编译工具链） | CUDA 11.8 |
| `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04` | 推理运行时（轻量） | CUDA 11.8 |

### 1.2 兼容性结论

**CUDA 11.8 完全满足需求，不必须使用 CUDA 12.1。** 理由如下：

| 依赖库 | CUDA 11.8 支持 | CUDA 12.1 支持 | 说明 |
|--------|---------------|---------------|------|
| PyTorch 2.0-2.4 | ✅ 完全支持 | ✅ 完全支持 | PyTorch 官方同时提供 cu118 和 cu121 wheel |
| ONNXRuntime 1.18 | ✅ 完全支持 | ✅ 完全支持 | ORT GPU 版支持 CUDA 11.8+ |
| TensorRT 8.6 | ✅ 完全支持 | ✅ 完全支持 | TensorRT 8.6 支持 CUDA 11.8+ |
| cuDNN 8.x | ✅ 已包含 | ✅ 已包含 | 两个镜像都含 cuDNN 8 |

### 1.3 已实施改进

- `train/Dockerfile` 已改为支持构建参数 `CUDA_BASE_IMAGE`，默认使用 CUDA 11.8
- 用户可通过 `--build-arg` 切换到 CUDA 12.1
- 两个镜像的分工建议：
  - `devel` 版 → 训练镜像（需要 nvcc 编译 CUDA kernel）
  - `runtime` 版 → 可用于自定义生产镜像（仅需 CUDA 运行时）

## 4. 新增AI能力的全链路原则（已写入规范）

已在 `docs/design/build_service.md` 第9节完整定义了14项全链路更新清单。核心原则：

> **每次新增AI能力模块时，必须同步更新所有关联子系统的代码、配置和Web页面。**

自动适配项（新增能力后无需改代码）：
- 训练 Web 页面（动态加载能力列表）
- 授权 Web 页面（动态读取能力列表）
- 编译 Web 页面（动态扫描能力目录）
- 生产 Web 页面（动态加载已部署能力）
- AI 编排系统（动态加载可用能力列表）

需手动更新项：
- C++ 能力插件源码
- 训练脚本（train.py, export.py）
- 测试推理器实现类
- Pipeline 配置文件（如需编排）
- 文档更新
